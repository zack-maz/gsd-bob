# Quick Task 260706-j81: Seed Bob's 270k context window

## Objective

Bob's runtime has a **270k context window**. Because gsd-bob degrades subagent work to
**sequential-inline execution** (the `BOB_CAPABILITY_DECL` lower-bound default — no isolated
subagents), the entire GSD loop shares one context window, so Bob's real 270k is the operative
budget. gsd-core keys its read-depth / advisory scaling on a top-level `context_window` integer
in `.planning/config.json` (default **200000** when absent; validated by `verify.cjs` W013).

Today the installer's `mergeTextMode` seeds only `workflow.text_mode: true`, so every Bob
install silently runs against the conservative 200k default. Seed Bob's real 270k so gsd-core's
budget math matches reality, and document the consideration.

## Tasks

### Task 1 — Extend the installer config merge to seed `context_window: 270000`

**Files:** `src/installer/config-merge.cjs`, `test/installer/config-merge.test.cjs`

**Action:**
- In `src/installer/config-merge.cjs`, add a named constant `BOB_CONTEXT_WINDOW = 270000`
  (with a short comment tying it to Bob's runtime window + the sequential-inline shared-context
  rationale). Keep the exported function name `mergeTextMode` (callers in `bin/gsd-bob.cjs` and
  tests must not change).

  hash.

  top-level `context_window` (it currently says "the SOLE text_mode guarantee").
- Preserve all existing discipline: never-clobber an unparseable config (still returns
  `{written:false, bytes:undefined}` without touching the file), non-object/array coercion,
  dryRun computes-but-does-not-write, and byte-identical idempotent re-runs.

**Update tests** in `test/installer/config-merge.test.cjs`:
