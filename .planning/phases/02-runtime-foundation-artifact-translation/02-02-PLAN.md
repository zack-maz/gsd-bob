<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-authored custom_modes.yaml → merge | hand-edited YAML (nested modes, multiline roleDefinition, comments) is parsed and re-emitted by the merge |
| GSD command/skill frontmatter → converter | source markdown frontmatter (possibly containing YAML flow chars) is reduced to Bob's whitelist and re-quoted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | Malicious/oversized `custom_modes.yaml` triggering a merge that drops or corrupts non-gsd user modes | mitigate | `yaml.load` (js-yaml v4 safe schema — does NOT execute arbitrary tags); slug-scoped filter touches ONLY `gsd`/`gsd-*`; merge.test.cjs asserts non-gsd modes preserved + idempotency. |
| T-02-05 | DoS (skill silently dropped) | Frontmatter injection — YAML flow chars in a description breaking Bob's parser so the skill is ignored | mitigate | Every emitted description/argument-hint passes through `yamlQuote` (= JSON.stringify); skill-golden.test.cjs asserts well-formed frontmatter. |
| T-02-06 | Elevation | Converter interpolating an unescaped frontmatter value into a shell string | accept | The converters emit markdown only; no shell interpolation occurs in conversion. The shell-out seam is `command`→`gsd_run` with args from GSD workflows, not raw user text. |
| T-02-07 | Tampering | Parity-first gap going SILENT — an unsupported skill emitted broken instead of flagged | mitigate | gateArtifact omits the artifact AND records a loud `unsupported on Bob: <reason>` roster line; unsupported-gate.test.cjs asserts the marker is present for the representative case. |
</threat_model>
