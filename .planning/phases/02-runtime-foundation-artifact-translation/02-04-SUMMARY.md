# Phase 2 Plan 04: Phase 2 Gap Closure (Bob body neutralization + loud-failure guards) Summary

Closes the Phase 2 verification BLOCKER and the two TRANS-04/05 WARNINGs by neutralizing `.claude`→`.bob` / `gsd:`→`gsd-` in both Bob converter bodies (the only peer converter that previously passed bodies through verbatim) and by hardening three silent-failure edges in `bob-adapter.cjs` to fail loud / exclude with a concrete reason — all under TDD with golden fixtures that fail on the old pass-through.

## What Was Built

### Task 1 — `convertClaudeToBobContent` + wired into both Bob converters (TRANS-01/02)
- Added `convertClaudeToBobContent(content, isGlobal=false)` to `runtime-artifact-conversion.cjs`, templated on `convertClaudeToAntigravityContent` (slash-before-bare replace ordering preserved) but mapping to the `.bob` home: global → `~/.bob`, local → `.bob`; always `./.claude/`→`./.bob/`, `.claude/`→`.bob/`, `gsd:`→`gsd-`. Pure string replaces, dependency-free, no `neutralizeAgentReferences` (backend-agnostic, RUNTIME-04). Exported in `module.exports`.
- `convertClaudeCommandToBobSkill` now neutralizes the FULL content before frontmatter reduction (mirrors the Antigravity skill converter); 5th positional renamed `_isGlobal`→`isGlobal` and used.
- `convertClaudeCommandToBobCommand` gained a 3rd optional `isGlobal=false` param; body is neutralized FIRST, THEN `$ARGUMENTS`→`$1` is projected (order matters).
- Frontmatter whitelists unchanged: skills still name+description-only, commands still description+argument-hint-only (no 02-02 regression).
- Commit: `3435622`

### Task 2 — Golden fixtures + tests exercise neutralization (WR-03)
- Skill/command `input.md` bodies now carry `@$HOME/.claude/...`, `~/.claude/...`, `.claude/skills/...`, and `/gsd:...` references; command input adds a `.claude/...` path and `/gsd:help`.
- `expected.md` for both regenerated by RUNNING the Task-1 converters (exact bytes, no hand-authoring → no whitespace drift).
- Added body-neutralization assertions to both golden tests: emitted output has NO `.claude` / NO `gsd:` and DOES contain `.bob` / `gsd-`; forbidden tokens built programmatically (`['.','claude'].join('')` etc.) so the test prose cannot self-trip.
- The byte-identity golden diff now FAILS on the old pass-through (expected.md carries `.bob`/`gsd-`) and PASSES only after Task 1 lands.
- Commit: `67f1338`

### Task 3 — Loud-failure guards in bob-adapter (TRANS-04/05)
- `mergeCustomModes`: a non-empty file that parses to a non-mapping root (scalar or sequence) throws a concrete, message-bearing error (`...root is not a mapping (got <type>); refusing to silently drop the gsd mode`). `null`/comment-only input still resolves to a fresh `{}` (empty-input happy path preserved).
- `gateArtifact`: candidate-shape guard added FIRST — `null`/`undefined`/non-string/empty name → `{supported:false, reason:'invalid candidate: missing or non-string name'}`. Never `{supported:true}` for a malformed candidate.
- `buildSupportRoster`: uses a `'<unnamed candidate>'` placeholder instead of interpolating a possibly-undefined name — never emits a malformed `undefined:` line.
- Added regression tests: sequence-root + scalar-root loud failure, empty/undefined happy path intact, `gateArtifact(null)` excluded, nameless candidate excluded, no `undefined:` roster line (token built programmatically).
- Commit: `35300dd`
