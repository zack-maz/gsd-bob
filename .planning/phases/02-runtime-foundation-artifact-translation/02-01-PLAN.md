| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `BOB_CONFIG_DIR` resolving to an attacker-chosen path (path traversal / absolute redirect) | accept | The env var is gsd-bob's own shim-level override set by the operating user; resolution reuses gsd-core's existing `resolveConfigHomeFromDescriptor` (no new path logic). Same trust model as `CLAUDE_CONFIG_DIR`/`CURSOR_CONFIG_DIR`. No untrusted network/remote source sets it. |
| T-02-02 | Tampering | Hand-edit to the generated `capability-registry.cjs` diverging from intended data | mitigate | Descriptor unit test asserts `configHome.name === '.bob'` + valid install axes; head-comment documents the generated-file divergence (vendored vs upstream-PR source) so the edit is intentional and reviewable. |
| T-02-03 | Information Disclosure | Model-backend brand leaking into core paths (backend-neutrality violation) | mitigate | `test/backend-neutrality.test.cjs` asserts zero model-name literals in the bob registry entry + adapter; CI gate. |
| T-02-SC | Tampering | npm install of `js-yaml` (supply chain) | mitigate | `js-yaml` pinned EXACT `4.1.0` (not the just-published 4.2.0); Package Legitimacy Audit verified genuine `nodeca/js-yaml` (254M dl/wk, 15-yr repo, no postinstall).
